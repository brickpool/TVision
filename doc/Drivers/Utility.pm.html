Keyboard support routines

Exported with :kbd.

ctrl_to_arrow
  func ctrl_to_arrow(Int|Str $key_code) : Int

This esoteric function converts certain control-key combinations to standard kbXXXX constant values.

The original WordStar word processor used a standard set of Ctrl-letter combinations as equivalents to IBM cursor movement keys and set a standard interpretation of these keys when used in word processing.

To assist in providing WordStar compatibility, ctrl_to_arrow can be used to convert $event->key_code values to their corresponding kbXXXX values.

The following chart shows the mapping from control keys to kbXXXX values.

  Control key   Lo(Keycode)   Maps to
  Ctrl-A        0x01          KB_HOME
  Ctrl-D        0x04          KB_RIGHT
  Ctrl-E        0x05          KB_UP
  Ctrl-F        0x06          KB_END
  Ctrl-G        0x07          KB_DEL
  Ctrl-S        0x13          KB_LEFT
  Ctrl-V        0x16          KB_INS
  Ctrl-X        0x18          KB_DOWN
get_alt_char
  func get_alt_char(Int $key_code) : Str

When $key_code specifies an Alt+Ch character combination, where Ch is a letter from 'A' to 'Z', get_alt_char extracts and returns the Ch character value.

For example,

  print get_alt_char( KB_ALT_A );

prints the single letter A.

The subroutine "get_alt_code" maps characters back to Alt+Ch combinations.

get_alt_code
  func get_alt_code(Str $ch) : Int

Maps a single character 'A' to 'Z' to the Keycode equivalent value for pressing Alt+Ch.

For example,

  get_alt_code(Str $ch);

returns 0x1e00, which is the value of KB_ALT_A.

The subroutine get_alt_code is the inverse function to "get_alt_char".

get_ctrl_char
  func get_ctrl_char(Int $key_code) : Str

Returns the ASCII character for the Ctrl+Key scancode that was given.

get_ctrl_code
  func get_ctrl_code(Str $ch) : Int

Returns the scancode corresponding to Ctrl+Ch key that is given.

String routines

Exported with :str.

format_str
  func format_str(Str $result, Str $format, @params)

format_str takes a string $format and a list of parameters in @params and produces a formatted string that is returned in Result.

format_str is typically used to insert parameter values into predefined strings, such as those used for error messages, or general program strings stored in a resource file.

The Format string contains both text and imbedded formatting information, as in this example:

  'File %s is %d bytes in size.'

The formatting characters %s and %d indicate that a string and a decimal value, respectively, should be substituted in these locations.

The values for the substitutions are specified in the Params parameter as shown in the example code, below.

$format specifiers have the form % [-] [nnn] X, where the brackets indicate optional items and X is a format character.

  Table of Format Specifiers
  %   Marks the beginning of a format specifier
  
  -   Indicates the items should be left justified (default is right justified)
  
  nnn Specifies the width of the result, where nnn is in the range from 0 to
      255.
      
      0 is equivalent to not specifying a width.
      
      If nnn is less than the width needed to display a particular item, the
      item is truncated to fit within the width value.
      
      For example, %3s allocates 3 character spaces. If the string parameter
      contains 'Turbo', then only the last 3 characters 'rbo' will be inserted
      into the result.
      
      If you use %-3s, then only the first 3 characters will be inserted, giving
      'Tur'.
  
  s   Format character indicating the parameter is a string pointer.
  
  d   Format character indicating the parameter is a Longint and is to be
      displayed in decimal.
  
  c   Format character specifying the low byte of the parameter is a character
      value.
  
  x   Format character specifying the parameter is a Longint to be displayed in
      hexadecimal.
  
  #   Resets the parameter index to the optional nnn value.

The @params parameter variable contains the data corresponding to each item specified in the $format string. There are two ways to pass the variable @params: 1) Pass the individual parameters comma separated, 2) Or use an array.

Note: This utility routine is for compatiblity only; please use the Perl built in function sprintf.

Buffer move routines

Exported with :move.

c_str_len
  func c_str_len(Str $s) : Int

Returns the length of control strings, which are any strings containing short-cut characters surrounded by tilde '~' characters, minus the number of tilde characters. For example,

  c_str_len( '~F~ile' );

has a length of 4.

move_buf
  func move_buf(ArrayRef $dest, ArrayRef $source, Int $attr, Int $count)

move_buf is typically used for copying text and video attribute to a TDrawBuffer-type array.

Such an array holds character bytes in the "low byte(s)" of each element and attribute values in the "high byte(s)".

move_buf copies $count elements from $source into the "low byte(s)" of the $dest destination parameter, setting each "high byte(s)" to the $attr value (or leaving the attribute as is if $attr equals zero).

See: "move_char", TDrawBuffer, TView->write_buf and TView->write_line.

move_c_str
  func move_c_str(ArrayRef $dest, Str $str, @attrs)

move_c_str copies a string to a TDrawBuffer array such that the text is alternately one of two different colors.

move_c_str copies the $str string parameter to the $dest (a TDrawBuffer array) and sets each character's attributes using either the first or second element of the @attrs array.

Initially, move_c_str uses the second element of @attrs, but upon encountering a "~" tilde character, move_c_str switches to the first element of @attrs.

Each tilde in the string causes move_c_str to toggle to the other @attrs attribute element.

move_c_str is used by Turbo Vision for setting up pulldown menu strings where the hot keys are set off in a different color from the rest of the text.

For example,

  new_sub_menu('~R~un', HC_NO_CONTEXT, new_menu( ...

You use move_c_str like this:

  my $a_buffer = [];
  ...
  move_c_str( $a_buffer, 'This ~is~ some text.', 0x07, 0x70 );
  $self->write_line( 10, 10, 18, 1, $a_buffer );

This sets the word 'is' to the attribute 0x07 and the rest of the text to 0x70.

See: TDrawBuffer, "move_char", "move_buf", "move_str", TView->write_buf and TView->write_line.

move_char
  func move_char(ArrayRef $dest, Str $c, Int $attr, Int $count)

Similar to "move_buf", except that this copies the single character $c, $count number of times, into each "low byte(s)" of the $dest parameter (which should be a TDrawBuffer type), and if $attr is non-zero, copies $attr to each "high byte(s)" position in the array of elements.

See: "move_buf", TDrawBuffer, TView->write_buf and TView->write_line.

move_str
  func move_str(ArrayRef $dest, Str $str, Int $attr)

move_str copies the $str string parameter to the $dest (a TDrawBuffer array) and sets each character's attributes to the video attribute contained in $attr.

See: TDrawBuffer, "move_char", "move_buf", "move_c_str", TView->write_buf and TView->write_line.
