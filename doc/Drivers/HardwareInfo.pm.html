NAME

TurboVision::Drivers::HardwareInfo - Turbo Vision Hardware driver

DESCRIPTION

THardwareInfo is a platform specific driver implementation that reads keystrokes and mouse events and emits output to a screen.

The interface definition was taken from the C++ Turbo Vision library.

Screen functions

Screen functions adopted from the C++ Turbo Vision library.

allocate_screen_buffer
  method allocate_screen_buffer() : ArrayRef

This function should initialize any data structures needed for the functionality of the driver, maybe do some allocation.

The function is guaranteed to be called only once; It can only be called again after a call to "free_screen_buffer".

The function "get_screen_rows" and "get_screen_cols" should be implemented correctly for a call of this function, as the allocate_screen_buffer call will initialize the video buffer array based on their values.

clear_screen
  method clear_screen(Int $w, Int $h)

If there is a faster way to clear the screen than to write spaces in all character cells, then it can be implemented here.

If the driver does not implement this function, then the general routines will write spaces in all video cells, and will call _update_screen(TRUE).

free_screen_buffer
  method free_screen_buffer()

This should clean up the video buffer data structures that have been initialized with the "allocate_screen_buffer" function.

The video buffer array will be cleared by the free_screen_buffer call.
